# Ask My CV — Backend

A RAG (Retrieval-Augmented Generation) API that answers questions about a CV/resume using LangChain, ChromaDB, and OpenAI.

## Stack

| Layer | Technology |
|---|---|
| API server | FastAPI + Uvicorn |
| RAG chain | LangChain (LCEL) |
| Embeddings & LLM | OpenAI |
| Vector database | ChromaDB (local) |
| PDF loader | PyPDF |

## Setup

### 1. Prerequisites

- Python 3.10+
- An OpenAI API key ([get one here](https://platform.openai.com/api-keys))

### 2. Create and activate the virtual environment

```bash
cd backend/
python -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

### 4. Configure environment variables

```bash
cp .env.example .env
```

Edit `.env` and fill in your `OPENAI_API_KEY` and the path to your CV file.

### 5. Add your CV

Place your PDF in the `data/` directory and update `CV_PATH` in `.env` to match the filename.

### 6. Index your CV (run once)

This reads the PDF, splits it into chunks, creates embeddings, and stores them in ChromaDB:

```bash
python -m app.ingest
```

You only need to re-run this when your CV changes.

### 7. Start the server

```bash
uvicorn app.main:app --reload --port 8000
```

### 8. Test the API

Open the interactive docs at `http://localhost:8000/docs` and use the `/chat` endpoint,
or use curl:

```bash
curl -X POST http://localhost:8000/chat \
     -H "Content-Type: application/json" \
     -d '{"question": "What programming languages do you know?"}'
```

## Project Structure

```
backend/
├── app/
│   ├── config.py     # loads .env variables
│   ├── prompts.py    # LangChain prompt template
│   ├── ingest.py     # one-time PDF → ChromaDB indexing
│   ├── rag.py        # RAG chain (retriever + LLM)
│   └── main.py       # FastAPI endpoints
├── data/             # put your CV PDF here
├── vectorstore/      # auto-generated by ingest.py (do not commit)
├── .env              # your secrets (do not commit)
├── .env.example      # template to share with others
└── requirements.txt
```
